# Midex_sales_dashboard
Интерактивный дашборд в Power BI для компании Midex. Проект включает в себя продвинутое моделирование данных (DAX), расчет KPI и прогнозирование выполнения планов продаж по 13 розничным филиалам.

## Описание визуализаций дашборда

Дашборд состоит из нескольких ключевых блоков, которые помогают всесторонне анализировать эффективность продаж:

*   **Ключевые показатели (KPI Cards - верхняя панель):**
    *   Мгновенно отображают главных лидеров и аутсайдеров по динамике продаж среди филиалов.
    *   Выделяют лучшего менеджера месяца для системы мотивации.
    *   Показывают общий тренд продаж (в %) и индикатор (Gauge chart) процента выполнения общего плана компании.

*   **Боковая панель навигации и фильтров (Left Sidebar):**
    *   Включает интерактивные срезы (Slicers) для фильтрации всех данных на странице по конкретному году и месяцу (например, Апрель 2026).
    *   Отображает мини-рейтинг ТОП-3 менеджеров по объемам продаж.

*   **Выполнение плана по филиалам (Левый центральный график):**
    *   Диаграмма типа "Пуля" (Bullet chart), которая наглядно визуализирует фактический процент выполнения плана по каждому из 13 складов/филиалов с цветовой индикацией успеха (зеленый/синий/красный).

*   **Прогнозирование продаж (Правый верхний график):**
    *   Столбчатая диаграмма "Цель и Ожидаемые продажи" сравнивает плановые показатели (Plan Target) с расчетным прогнозом продаж (Sales Forecast) в разрезе каждого филиала, помогая предвидеть кассовые разрывы или перевыполнение.

*   **Детализированная матрица KPI (Правая нижняя таблица):**
    *   Подробная таблица эффективности каждого менеджера. Включает такие метрики, как: процент выполнения плана, количество заказов, привлеченные новые клиенты, отгрузки и итоговый **Общий балл KPI** (комплексная оценка эффективности).

    ### 1. Расчет комплексного балла менеджера (Общий балл KPI)
Для объективной оценки работы сотрудников разработана взвешенная система скоринга, учитывающая выполнение плана, количество заказов и отгрузок. При этом за каждого привлеченного нового клиента менеджер получает фиксированный 1 балл.

```dax
Общий балл KPI = 
VAR PlanWeight = 0.50
VAR OrderWeight = 0.20
VAR ShipmentsWeight = 0.20

// Расчет баллов по плану (процент выполнения переводится в баллы с учетом веса)
// Пример: 133% выполнения = 133 * 0.50 = 66.5 баллов
VAR PlanScore = [Вып. плана %] * 100 * PlanWeight 

// Расчет баллов за заказы и отгрузки (количество умножается на вес)
VAR OrderScore = [Кол-во заказов] * OrderWeight
VAR ShipmentsScore = [Отгрузки] * ShipmentsWeight

// За каждого нового клиента дается ровно 1 балл
VAR NewClientsScore = [Новые клиенты] * 1

RETURN
    PlanScore + OrderScore + ShipmentsScore + NewClientsScore

2. Динамическое прогнозирование продаж (Sales Forecast)
Для каждого из 13 филиалов рассчитывается ожидаемый объем продаж к концу месяца (run rate) на основе текущей динамики и исторических данных. Это позволяет заранее выявлять филиалы, рискующие не выполнить план.
    Sales Forecast = 
VAR CurrentDays = DAY(MAX('Calendar'[Date]))
VAR TotalDaysInMonth = DAY(EOMONTH(MAX('Calendar'[Date]), 0))
VAR CurrentSales = [Total Sales]

// Расчет средней скорости продаж в день
VAR DailyRunRate = DIVIDE(CurrentSales, CurrentDays, 0)
VAR Forecast = CurrentSales + (DailyRunRate * (TotalDaysInMonth - CurrentDays))

RETURN
    IF(
        ISBLANK(CurrentSales),
        BLANK(),
        Forecast
    )
```

## Страница 2: Детальная информация по складу (Store Info)

Данная страница предназначена для глубокого (drill-down) анализа конкретного филиала. Она позволяет руководителям и менеджерам отслеживать ежедневную динамику и состояние складских запасов:

*   **Карточки показателей склада (Верхняя панель):**
    *   Отображают название выбранного филиала (например, Самарканд), а также ключевые метрики за период: количество отгрузок, заказов, новых клиентов и наиболее продаваемую категорию товаров (Топ Категория).

*   **Ежедневная динамика продаж (Левый верхний график):**
    *   Линейный график продаж по дням с наложенной линией тренда. Позволяет визуально оценивать стабильность продаж, выявлять пиковые дни и спады активности в течение месяца.

*   **Топ-10 клиентов (Правая верхняя таблица):**
    *   Рейтинг ключевых покупателей выбранного филиала по объему продаж и количеству отгрузок. Помогает фокусироваться на VIP-клиентах.

*   **Продажи по категориям (Левый нижний график):**
    *   Горизонтальная линейчатая диаграмма, демонстрирующая распределение выручки по различным товарным группам (бронь, запчасти, сальники и т.д.), что помогает понять структуру спроса.

*   **Анализ складских остатков и оборачиваемости (Правая нижняя таблица):**
    *   Критически важный отчет для управления запасами. Включает наименование товара, текущий остаток на складе, историю продаж (за 90 дней) и расчетный показатель **«Запас на (дней)»**. Этот функционал помогает предотвращать дефицит товаров (Out-of-Stock) и своевременно планировать закупки.


    ### 3. Анализ оборачиваемости запасов (Run Rate / Запас в днях)
Для предотвращения дефицита товаров (Out-of-Stock) и оптимизации закупок рассчитывается показатель, показывающий, на сколько дней хватит текущего остатка на складе. Расчет базируется на средней скорости продаж товара за последние 90 дней.

```dax
Запас на (дней) = 
// 1. Берем общее количество проданного товара за последние 90 дней
VAR SalesLast90Days = [Продажи 90 дн.] 

// 2. Вычисляем среднюю скорость продаж в день (Daily Run Rate)
VAR DailyRunRate = DIVIDE(SalesLast90Days, 90, 0)

// 3. Текущий остаток на складе
VAR CurrentStock = [Остаток]

// 4. Расчет количества дней
VAR DaysOfInventory = DIVIDE(CurrentStock, DailyRunRate, 0)

RETURN
    IF(
        CurrentStock > 0 && DailyRunRate == 0,
        9999, // Индикатор залежалого товара (остаток есть, но продаж не было)
        IF(ISBLANK(CurrentStock) || CurrentStock == 0, 0, ROUND(DaysOfInventory, 0))
    )
